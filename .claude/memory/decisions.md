# Architecture Decision Records (ADR)

> 주요 아키텍처 결정 기록

## ADR-001: HTTP 배치 모드 STT

**날짜**: 2026-02-01
**상태**: 채택됨

### 컨텍스트
브라우저에서 RTZR WebSocket 스트리밍 연결이 필요하지만, 브라우저 WebSocket은 Authorization 헤더를 지원하지 않음.

### 결정
HTTP 배치 모드 사용 (8초마다 오디오 청크 전송)

### 대안
1. 서버 측 WebSocket 프록시 (복잡도 증가)
2. subprotocol 인증 (RTZR 미지원)

### 결과
- 장점: 구현 간단, 안정적
- 단점: 실시간성 약간 감소 (8초 지연)

## ADR-002: KMS 비디오 프록시

**날짜**: 2026-02-01
**상태**: 채택됨

### 컨텍스트
KMS 비디오 서버가 CORS를 허용하지 않아 브라우저에서 직접 재생 불가.

### 결정
Next.js API Route를 통한 프록시 구현 (`/api/kms/proxy`)

### 대안
1. CORS 허용 요청 (외부 서버라 불가)
2. iframe 사용 (제어 불가)

### 결과
- Range 헤더 지원으로 비디오 시킹 가능
- 프록시 지연 최소화

## ADR-003: 문장 단위 자막 버퍼링

**날짜**: 2026-02-01
**상태**: 채택됨

### 컨텍스트
STT 결과가 짧은 청크로 나뉘어 자막이 파편화됨.

### 결정
마침표/물음표/느낌표로 끝날 때까지 버퍼링 후 하나의 자막으로 표시

### 결과
- 가독성 향상
- 타임스탬프 정확도 향상 (문장 시작~끝 시간)
